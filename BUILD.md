# Как собирал Optimus AI и почему так

## Задача

Нужно рабочее **ядро агента**, которое ведёт рекламу по метрикам: CPC, показы, клики, конверсии, CPA. Не идеальный продукт под все площадки, а прототип, который можно потрогать и понять логику.

## Почему Python + FastAPI

- Быстро собрать CLI, API и дашборд в одном репозитории
- Pydantic для типов кампаний и метрик
- SQLite без отдельной инфраструктуры — достаточно для прототипа
- Typer + Rich для удобного CLI в терминале

## Почему Adapter Pattern

Все площадки (Яндекс, Google, Telegram, Instagram) сводятся к одному контракту:

- `get_period_metrics()` — что случилось за период
- `set_bid()` — изменить ставку
- `pause()` / `resume()`
- `advance_tick()` — шаг времени

Сегодня за адаптером стоит `MarketSimulator`, завтра — HTTP-клиент к API. Агент не меняется.

## Почему Rules + Bandit, а не LLM для ставок

LLM хорош для объяснений и креатива, но **плох как детерминированный оптимизатор ставок**:

- Нет гарантии соблюдения бюджета и CPA
- Дорого вызывать на каждый час
- Сложно тестировать и воспроизводить

Поэтому:

1. **Rule Engine** — здравые маркетинговые правила (CPA выше target → снижаем ставку; CPA хороший и мало показов → наращиваем)
2. **Thompson Sampling bandit** — запоминает, какие из допустимых действий (`-20%`, `-10%`, `0%`, `+5%`, `+10%`) давали лучший reward (низкий CPA)

Это «самообучение по метрикам» без тяжёлого ML.

## Как устроен мок-рынок

`MarketSimulator` имитирует:

| Фактор | Эффект |
|--------|--------|
| Ставка | Позиция в аукционе → больше показов |
| S-кривая CTR | Высокая позиция → выше CTR |
| Час суток | Пик 10–14 и 18–21, ночь — спад |
| День недели | Выходные слабее (B2B) |
| Конкуренция | Высокая ставка → дороже CPC |
| Шоки (~4%/час) | Внезапный рост конкуренции |
| Усталость креатива | CTR падает после флага агента |

Агент видит шум и задержку — как в реальной рекламе нельзя реагировать на 3 клика.

## Поток данных

```
тик N:
  1. Simulator генерирует impressions/clicks/conversions/spend
  2. KPI: CTR, CPC, CR, CPA, ROAS
  3. Rules предлагают кандидатов
  4. Bandit выбирает действие
  5. Ставка меняется в адаптере
  6. Bandit обновляется по reward
  7. Всё пишется в SQLite
```

## Как расширить на Google Ads / Telegram

1. Создать `adapters/google.py` implements `AdPlatformAdapter`
2. В `AgentLoop.get_adapter()` добавить ветку `platform == "google"`
3. Подключить OAuth и Reports API площадки
4. Маппинг их метрик в `PeriodMetrics`

Ядро (`AgentLoop`, `OptimizerEngine`, `MetricsStore`) остаётся без изменений.

## Что сознательно не делал

- Реальные API-ключи (stub `YandexDirectAdapter` с инструкцией)
- A/B креативов (только флаг fatigue в симуляторе)
- LLM-генерация текстов объявлений
- Мультиплатформенный микс бюджета
